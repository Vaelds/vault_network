STATUS REPORT // NETWORK SNAPSHOT

Projektstatus

Vault Net er opbygget som et fælles terminalsæt med samme header på tværs af siderne, data-attributter til de interaktive kontroller og særskilte CSS-lag til bunker-, alarm- og Matrix-udtryk. 404-siden fungerer som implementeroversigt for de centrale moduler, så man kan se både funktion og placering i filstrukturen.

  • Topmenu med specialknapper, hover-respons og page fade mellem interne sider
  • Matrix mode med vedvarende state, baggrundslag og genereret code rain
  • Popup-artikler der åbner fra kortenes position og fyldes dynamisk via JavaScript
  • Vedvarende Spotify-spor med popup-spiller, statusdeling og direkte Spotify-link
  • Eksisterende hotspot-, countdown-, spotlight- og formularfunktioner i samme fælles system

NAVIGATION LAYER // HEADER CONTROLS

Topmenu og særlige knapper

Headeren er bygget ens på alle sider med samme nav-container, samme links og en samlet .nav-actions-blok. Det gør, at Matrix-knappen, spotlight-kontrollen, alarmknappen og burgerknappen kan genbruges uden særskilt markup fra side til side.

De interne sideskift håndteres ikke af CSS alene. I js/main.js opfanges klik på interne links, body får klassen is-page-transitioning, og i css/fallout.css kører der en kort fade-out, før næste side åbnes. På den måde bliver navigationen en kontrolleret terminalovergang i stedet for et hårdt hop.

Matrix-knappen er ikke et ikonbillede. I css/matrix-theme.css tegnes selve knappen med gradienter og scanline-lag, .matrix-toggle__glyph::before skriver bogstavet M, og ::after laver den lille lysprik, som kredser rundt om symbolet med @keyframes matrixDotOrbit. Hover-effekten er nu også eksplicit genskabt i Matrix mode, så knappen igen skalerer og lyser op ved roll over.

MATRIX STATE // MODE SWITCH

Matrix mode

Selve tilstanden styres i js/matrix-theme.js. Når en knap med data-matrix-toggle klikkes, skiftes body-klassen matrix-theme, og valget gemmes i localStorage under nøglen vault-matrix-theme. Derfor følger Matrix mode med videre, når brugeren går mellem sider.

Ved hvert state-skift synkroniseres også knappernes aria-pressed, titeltekst og label, så både desktop- og mobilknappen afspejler samme status. Den samme funktion opdaterer samtidig lydpanelet, så Spotify-docken automatisk skifter mellem bunker-look og Matrix-look uden separat sidekode.

BACKGROUND STACK // MATRIX SCENE

Hvordan Matrix-baggrunden er lavet

Matrix-baggrunden består af flere lag. Først farver body.matrix-theme hele siden med mørke grønne gradienter. Derefter lægger body.matrix-theme::before scanlines og en svag tone ovenpå, mens body.matrix-theme::after tilføjer vignette, så kanterne bliver mørkere og midten holdes mere læsbar.

Selve den faldende kode er et separat DOM-lag og ikke et baggrundsbillede. ensureRainLayer() i js/matrix-theme.js opretter .matrix-code-rain med et slør-lag og en container til kolonnerne. Når laget bygges første gang, genereres der span-kolonner med tilfældige tegn fra MATRIX_SYMBOLS, forskellig længde, forskellig hastighed, forskellig opacitet og et lille vandret drift-offset.

I css/matrix-theme.css animeres hver kolonne med @keyframes matrixCodeFall, så teksten falder fra over skærmen til langt under viewporten. Regnlaget ligger på z-index: 0, mens header, main og footer løftes til z-index: 1, så effekten bliver bagved indholdet i stedet for ovenpå det.

ARCHIVE CARDS // POPUP ARTICLES

Popup-artiklerne på forsiden

Artikelkortene på forsiden er knapper og ikke almindelige links. Hvert kort har et data-article-id, og markup til selve modalvinduet ligger allerede i index.html. Indholdet hentes ikke udefra, men fyldes ind fra articleLibrary i js/main.js, hvor titel, kicker, stempel, billede og HTML-tekst er samlet pr. artikel.

Når en artikel åbnes, flyttes modalen op under body, indholdet indsættes, scroll nulstilles og fokus gemmes. JavaScript måler derefter centerpunktet på det kort, der blev klikket på, og skriver forskellen til CSS-variablerne --article-open-offset-x og --article-open-offset-y. Derfor ser popupen ud til at vokse ud fra det valgte kort i stedet for bare at poppe ind midt på skærmen.

Lukning sker via backdrop, krydsknap eller Escape. Samtidig fjernes body.article-modal-open, footeren skjules kun mens modalen er åben, og fokus sendes tilbage til det kort, der åbnede artiklen. Selve udtrykket er styret i css/article-popups.css, hvor både plakatkort, modalpanel, scrollområde og mobiljusteringer er samlet.

AUDIO LAYER // PERSISTENT TRACK

Spotify-link og vedvarende musik

Lydpanelet bliver oprettet dynamisk i js/matrix-theme.js som .matrix-audio-dock. Panelet indeholder en Play/Pause-knap, en Spiller-knap og et direkte Spotify-link. Spotify-linket er en almindelig ekstern <a>-reference til det konkrete track på open.spotify.com, så brugeren kan åbne nummeret direkte i Spotify eller browseren uden at bruge popup-spilleren.

Selve den vedvarende afspilning foregår i det separate vindue matrix-audio-player.html. Når der trykkes på Play, åbnes popupen, og js/matrix-audio-player.js loader Spotify IFrame API og opretter afspilleren med spotify:track:1ZsUmKQt3RvaxuHVHbJlyI. Fordi spilleren ligger i sit eget vindue, stopper nummeret ikke bare, når brugeren skifter side på sitet.

Status deles mellem sider og popup via BroadcastChannel og localStorage med nøglen vault-matrix-audio-playing. Det betyder, at Play/Pause-teksten, statuslinjen og mode-skiftet kan holdes synkroniseret. Spiller-knappen åbner eller fokuserer popup-vinduet, mens Spotify-linket alene er genvejen til tracket.

CONTROL LAYER // LIGHT AND ALERT

Spotlight og alarm i samme kontrolsystem

Spotlighten oprettes lazily i js/main.js som en separat .spotlight-overlay. Markørens position skrives til CSS-variablerne --spotlight-x og --spotlight-y via requestAnimationFrame, og overlayet bruger en radial gradient til at simulere lyskeglen. På coarse pointer eller reduceret bevægelse falder systemet tilbage til en fast spotlight-placering.

Alarmtilstanden bruger samme data-attributbaserede knapmønster som spotlighten. Klassen alarm-active aktiverer baggrundsoverlay og rødt udtryk, mens lyden genereres direkte i browseren med Web Audio API i stedet for at hente en ekstern lydfil. På mobil udskiftes alarmknappen samtidig visuelt med panic-knap-grafikken gennem Matrix-CSS.
